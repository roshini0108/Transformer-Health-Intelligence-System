## 🎯 Problem Statement

Electricity distribution transformers fail without warning, causing:

- **Unplanned outages** affecting 200–500 consumers per failure
- **Emergency repair costs** of ₹1.8–20 lakhs per transformer depending on capacity
- **Revenue loss** from unmetered supply during outage hours
- **Strain on field teams** who operate reactively instead of proactively

APEPDCL, like most Indian power distribution utilities, currently relies on **reactive maintenance** — transformers are inspected only after a consumer complaint or failure occurs. This project replaces that with **predictive maintenance**, where AI continuously monitors transformer health and flags at-risk units weeks before they fail.

This work is directly aligned with the Government of India's ₹3.03 lakh crore RDSS (Revamped Distribution Sector Scheme), which mandates AT&C loss reduction and infrastructure modernisation for all DISCOMs.

---

## 💡 Solution Overview

The system continuously analyses transformer sensor data (oil temperature, load, power factor, harmonic distortion, voltage) and computes a **0–100 health score** for every transformer, updated daily. Engineers see:

1. A **dashboard** ranking all transformers by risk level (LOW / MEDIUM / HIGH / CRITICAL)
2. A **failure probability** for the next 30 days, generated by a trained XGBoost model
3. **Automatic alerts** when thresholds are breached (overload, overheating, anomalies)
4. **Explainable AI** — the dashboard shows *why* a transformer was flagged, not just that it was

## 📊 Key Results

| Metric | Result |
|---|---|
| XGBoost ROC-AUC Score | **0.9976** |
| Recall (failures caught) | **84%** |
| Precision (alert accuracy) | **75.9%** |
| Transformers monitored | **50** |
| Sensor readings processed | **423,360** |
| Pre-programmed failures correctly identified | **8 / 8 (100%)** |
| Top predictive feature | **Power factor trend** (61.6% importance) |

**Key finding:** Power factor deterioration trend emerged as the single strongest predictor of transformer failure — outperforming oil temperature, the conventionally assumed top indicator. This suggests power factor monitoring can provide earlier warning than thermal monitoring alone.

## 🛠 Tech Stack

**Backend**
- Python 3.11, FastAPI, Uvicorn
- PostgreSQL 16, SQLAlchemy ORM
- Pydantic for request/response validation

**Machine Learning**
- XGBoost (failure prediction classifier)
- Isolation Forest (unsupervised anomaly detection)
- scikit-learn, pandas, NumPy
- joblib for model persistence

**Frontend**
- React 18 + Vite
- React Router for navigation
- Recharts for time-series visualisation
- Axios for API communication

**Data**
- 423,360 synthetic sensor readings (hourly, 1 year, 50 transformers)
- Modelled on real Visakhapatnam district transformer locations
- Realistic seasonal patterns (April–June peak load, AP climate)

## ⚙️ Setup Instructions

### Prerequisites
- Python 3.11+
- Node.js 20+
- PostgreSQL 16+

### 1. Clone and install backend dependencies
```bash
pip install -r backend/requirements.txt
```

### 2. Configure environment
Create `backend/.env`:
```
DATABASE_URL=postgresql://postgres:YOUR_PASSWORD@localhost:5432/apepdcl_transformer
```

### 3. Generate training data and train models
```bash
python ml/src/data_generator.py
python ml/src/preprocessor.py
python ml/src/anomaly_model.py
python ml/src/prediction_model.py
```

### 4. Seed database and import data
```bash
python backend/seed.py
python backend/import_readings.py
python backend/run_predictions.py
```

### 5. Start the backend
```bash
cd backend
uvicorn main:app --reload
```
API docs available at `http://localhost:8000/docs`

### 6. Start the frontend
```bash
cd frontend
npm install
npm run dev
```
Dashboard available at `http://localhost:5173`

## 🧠 Machine Learning Details

**Feature engineering** (19 features per reading):
- Raw sensor values: load %, oil temp, ambient temp, power factor, harmonic distortion
- Rolling averages: 24-hour and 7-day windows
- Trend features: 7-day mean minus 30-day mean (captures direction of change)
- Stress counters: overload events, high-temperature events, fault events in last 7 days

**Isolation Forest** (anomaly detection):
- Trained only on healthy readings to learn "normal" behaviour
- Flags readings that deviate from learned patterns
- Unsupervised — requires no failure labels

**XGBoost** (failure prediction):
- Binary classifier: will this transformer fail within 30 days?
- `scale_pos_weight=77` to handle severe class imbalance (1.4% failure rate)
- 500 estimators, max depth 6, learning rate 0.05

**Health Scorer** (final output):
- Combines ML penalty (anomaly score + failure probability) with rule-based penalties (IEEE/IEC threshold violations)
- Outputs: 0–100 score, risk level, contributing factors, threshold violations
